# UI 수정 요청 템플릿 (Cursor용)

아래 템플릿을 그대로 복붙해서 요청하면, UI 디테일 작업 정확도가 높아집니다.

---

## 템플릿

```text
[작업]
<무엇을 수정할지 한 줄로>

[대상 파일]
- <파일1>
- <파일2>

[고정 조건]
- <절대 바꾸면 안 되는 것 1>
- <절대 바꾸면 안 되는 것 2>
- <반드시 지켜야 하는 시각 규칙 1>
- <반드시 지켜야 하는 시각 규칙 2>

[수치]
- <아이콘 크기>
- <간격>
- <라운딩>
- <테두리 두께>

[완료 기준(반드시 충족)]
1) <상태 A에서 시각 일관성 조건>
2) <상태 B에서 시각 일관성 조건>
3) <정렬 조건>
4) <텍스트/아이콘 배치 조건>
5) 변경 파일/변경 항목만 짧게 보고
```

---

## LabUI 뽑기 버튼 예시 (현재 프로젝트용)

```text
[작업]
LabUI 뽑기 버튼만 수정.

[대상 파일]
- LabUI.tscn
- (필요시) LabUI.gd

[고정 조건]
- 버튼 크기 변경 금지
- 아이콘은 버튼 중앙 정렬
- 텍스트 "뽑기"는 아이콘 아래
- 아이콘 테두리 금지
- 버튼 테두리만 유지
- hover 효과는 반드시 보이게 (색상 변화 명확)
- hover/focus/pressed 라운딩 동일(깨짐 금지)

[수치]
- 아이콘 56x56
- 아이콘-텍스트 간격 6
- 버튼 라운딩 12
- 버튼 테두리 2px

[완료 기준(반드시 충족)]
1) 기본/hover/pressed 상태에서 외곽선 모양 동일
2) hover 시 배경색 변화가 한눈에 보임
3) 아이콘이 좌우 중심에서 1px 이상 안 벗어남
4) 뽑기 텍스트가 아이콘 아래 중앙
5) 변경 파일/변경 항목만 짧게 보고
```

---

## 요청 팁

- "예쁘게" 대신 숫자(크기/간격/라운딩)를 꼭 같이 적기
- "유지"할 기준 파일을 명시하기 (예: `AuthUI.tscn` 스타일 유지)
- 완료 기준을 3~5개 체크 항목으로 적기
